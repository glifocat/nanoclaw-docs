The message router handles formatting messages for agents and routing outbound messages to the correct channel.

## Core functions

### escapeXml

Escapes special XML characters in strings.

```typescript
function escapeXml(s: string): string
```

<ParamField path="s" type="string" required>
  String to escape
</ParamField>

<ResponseField name="return" type="string">
  XML-escaped string with `&`, `<`, `>`, and `"` replaced with entities
</ResponseField>

**Example:**

```typescript
import { escapeXml } from './router.js';

const escaped = escapeXml('Hello <world> & "friends"');
// 'Hello &lt;world&gt; &amp; &quot;friends&quot;'
```

### formatMessages

Formats an array of messages as XML for agent consumption.

```typescript
function formatMessages(messages: NewMessage[], timezone: string): string
```

<ParamField path="messages" type="NewMessage[]" required>
  Array of messages to format
</ParamField>

<ParamField path="timezone" type="string" required>
  IANA timezone identifier (e.g., `"America/New_York"`). Used to format timestamps in local time and included in the output `<context>` header.
</ParamField>

<ResponseField name="return" type="string">
  A `<context>` header followed by XML-formatted messages wrapped in `<messages>` tags
</ResponseField>

**Output format:**

```xml
<context timezone="America/New_York" />
<messages>
<message sender="John Doe" time="2:34 PM">Hello!</message>
<message sender="Jane Smith" time="2:35 PM" reply_to="msg-001">
  <quoted_message from="John Doe">Hello!</quoted_message>Hi there!</message>
</messages>
```

When a message includes reply context (`reply_to_message_id`), a `reply_to` attribute is added to the `<message>` element. If `reply_to_message_content` and `reply_to_sender_name` are also present, a nested `<quoted_message>` element is included with the original sender and content.

**Example:**

```typescript
import { formatMessages } from './router.js';

const messages = [
  {
    id: '1',
    chat_jid: '123@g.us',
    sender: '456@s.whatsapp.net',
    sender_name: 'John',
    content: 'Hello!',
    timestamp: '2026-02-28T12:34:56.789Z',
  },
  {
    id: '2',
    chat_jid: '123@g.us',
    sender: '789@s.whatsapp.net',
    sender_name: 'Jane',
    content: 'Hi there!',
    timestamp: '2026-02-28T12:35:00.000Z',
    reply_to_message_id: '1',
    reply_to_message_content: 'Hello!',
    reply_to_sender_name: 'John',
  },
];

const formatted = formatMessages(messages, 'America/New_York');
```

### stripInternalTags

Removes `<internal>...</internal>` blocks from agent output.

```typescript
function stripInternalTags(text: string): string
```

<ParamField path="text" type="string" required>
  Text containing potential `<internal>` tags
</ParamField>

<ResponseField name="return" type="string">
  Text with all `<internal>` blocks removed and trimmed
</ResponseField>

**Purpose:**

Agents use `<internal>` tags for internal reasoning that should not be shown to users.

**Example:**

```typescript
import { stripInternalTags } from './router.js';

const raw = `<internal>Analyzing request...</internal>Here's your answer!`;
const clean = stripInternalTags(raw);
// "Here's your answer!"
```

### formatOutbound

Prepares agent output for sending to users. Strips internal tags and optionally converts Markdown to the target channel's native text syntax.

```typescript
function formatOutbound(rawText: string): string
```

<ParamField path="rawText" type="string" required>
  Raw text from agent
</ParamField>

<ResponseField name="return" type="string">
  Cleaned text ready for user display (empty string if no content after stripping)
</ResponseField>

**Example:**

```typescript
import { formatOutbound } from './router.js';

// Without channel formatting
const raw = `<internal>Debug info</internal>Response to user`;
const outbound = formatOutbound(raw);
// "Response to user"

// With channel formatting (requires /channel-formatting skill)
const formatted = formatOutbound(raw, 'whatsapp');
// Markdown markers converted to WhatsApp-native syntax
```

<Note>
The `channel` parameter is added by the `/channel-formatting` skill (`skill/channel-formatting` branch). On base NanoClaw without the skill applied, `formatOutbound` only accepts `rawText`.
</Note>

### routeOutbound

Routes a message to the appropriate channel.

```typescript
function routeOutbound(
  channels: Channel[],
  jid: string,
  text: string,
): Promise<void>
```

<ParamField path="channels" type="Channel[]" required>
  Array of available channels
</ParamField>

<ParamField path="jid" type="string" required>
  Chat JID to send to
</ParamField>

<ParamField path="text" type="string" required>
  Text to send
</ParamField>

**Throws:** Error if no connected channel owns the JID

**Example:**

```typescript
import { routeOutbound } from './router.js';

await routeOutbound(channels, '123@g.us', 'Hello!');
```

### findChannel

Finds the channel that owns a given JID.

```typescript
function findChannel(
  channels: Channel[],
  jid: string,
): Channel | undefined
```

<ParamField path="channels" type="Channel[]" required>
  Array of available channels
</ParamField>

<ParamField path="jid" type="string" required>
  Chat JID to look up
</ParamField>

<ResponseField name="return" type="Channel | undefined">
  Channel that owns the JID, or `undefined` if none found
</ResponseField>

**Example:**

```typescript
import { findChannel } from './router.js';

const channel = findChannel(channels, '123@g.us');
if (channel) {
  await channel.sendMessage('123@g.us', 'Hello!');
}
```

## Types

### NewMessage

```typescript
interface NewMessage {
  id: string;                       // Unique message ID
  chat_jid: string;                 // Chat identifier (e.g., "123@g.us")
  sender: string;                   // Sender JID (e.g., "456@s.whatsapp.net")
  sender_name: string;              // Display name
  content: string;                  // Message text
  timestamp: string;                // ISO 8601 timestamp
  is_from_me?: boolean;             // Whether sent by the bot
  is_bot_message?: boolean;         // Whether this is a bot message
  thread_id?: string;               // Thread/topic ID (channel-specific)
  reply_to_message_id?: string;     // ID of the message being replied to
  reply_to_message_content?: string; // Content of the quoted message
  reply_to_sender_name?: string;    // Display name of the quoted message sender
}
```

The `thread_id` and `reply_to_*` fields allow any channel to pass quoted message context to the agent. These fields are optional — channels that don't support replies simply omit them.

### Channel

```typescript
interface Channel {
  name: string;
  connect(): Promise<void>;
  sendMessage(jid: string, text: string): Promise<void>;
  isConnected(): boolean;
  ownsJid(jid: string): boolean;
  disconnect(): Promise<void>;
  setTyping?(jid: string, isTyping: boolean): Promise<void>;   // Optional
  syncGroups?(force: boolean): Promise<void>;                   // Optional
}
```
