NanoClaw configuration is managed through environment variables, the `.env` file, and the `src/config.ts` module. In v2, some configuration has moved to `container.json` per agent group.

## Environment variables

Configuration is read from `.env` file or `process.env`, with hardcoded fallbacks.

<ParamField path="ASSISTANT_NAME" type="string" default="Andy">
  Name of the assistant. Used in trigger pattern and message routing.
</ParamField>

<ParamField path="ASSISTANT_HAS_OWN_NUMBER" type="boolean" default="false">
  Whether the assistant has its own phone number or dedicated account. Set to `"true"` to enable.
</ParamField>

<ParamField path="CONTAINER_IMAGE" type="string" default="nanoclaw-agent:latest">
  Docker image to use for agent containers. Per-agent-group images are tagged as `nanoclaw-agent:{agentGroupId}`.
</ParamField>

<ParamField path="CONTAINER_TIMEOUT" type="number" default="1800000">
  Container timeout in milliseconds (default: 30 minutes).
</ParamField>

<ParamField path="CONTAINER_MAX_OUTPUT_SIZE" type="number" default="10485760">
  Maximum container output size in bytes (default: 10 MB).
</ParamField>

<ParamField path="IDLE_TIMEOUT" type="number" default="1800000">
  How long to keep container alive after last result in milliseconds (default: 30 minutes).
</ParamField>

<ParamField path="ONECLI_URL" type="string" default="http://localhost:10254">
  URL for the OneCLI Agent Vault that handles credential injection for containers.
</ParamField>

<ParamField path="ONECLI_API_KEY" type="string">
  API key for authenticating with the OneCLI Agent Vault. Optional — only needed if your OneCLI instance requires authentication. Read by the OneCLI SDK at request time. Setup writes this automatically when you pass `--onecli-api-token` (or `NANOCLAW_ONECLI_API_TOKEN`) for a remote vault.
</ParamField>

<ParamField path="MAX_MESSAGES_PER_PROMPT" type="number" default="10">
  Maximum number of messages sent to container agents per prompt.
</ParamField>

<ParamField path="MAX_CONCURRENT_CONTAINERS" type="number" default="5">
  Maximum number of concurrent agent containers.
</ParamField>

<ParamField path="LOG_LEVEL" type="string" default="info">
  Logging verbosity. Valid values: `debug`, `info`, `warn`, `error`, `fatal`.
</ParamField>

<ParamField path="TZ" type="string" default="system timezone">
  Timezone for scheduled tasks (cron expressions). Resolved from `TZ` env, `.env` file, then system default. Validated as a real IANA timezone identifier. Falls back to `UTC` if no valid timezone is found.
</ParamField>

## Setup-time environment variables

These variables are read by `nanoclaw.sh` and `setup:auto` to pre-fill answers in the installer's advanced-settings screen. They map 1:1 to CLI flags (see [Advanced setup flags](/installation#advanced-setup-flags)) and are folded into `process.env` before any setup step runs.

<ParamField path="NANOCLAW_ONECLI_API_HOST" type="string" default="https://app.onecli.sh">
  Use a remote OneCLI vault instead of installing one locally. When set, the OneCLI setup step skips the local-vs-fresh prompt, health-checks the URL, and runs the CLI installer in remote-only mode (writes `ONECLI_URL` to `.env`). Equivalent flag: `--onecli-api-host`.
</ParamField>

<ParamField path="NANOCLAW_ONECLI_API_TOKEN" type="string">
  Bearer token for the remote OneCLI vault. Required if `NANOCLAW_ONECLI_API_HOST` points at an authenticated gateway. Must start with `oc_`. Setup runs `onecli auth login --api-key` so CLI calls during the rest of setup are authenticated, and writes the value to `.env` as `ONECLI_API_KEY` for the runtime SDK. Equivalent flag: `--onecli-api-token`.
</ParamField>

<ParamField path="NANOCLAW_ANTHROPIC_BASE_URL" type="string">
  Use a proxy or alternative endpoint instead of `api.anthropic.com`. Equivalent flag: `--anthropic-base-url`.
</ParamField>

<ParamField path="NANOCLAW_DISPLAY_NAME" type="string">
  Skip the "what should your assistant call you?" prompt by setting the display name up front. Equivalent flag: `--display-name`.
</ParamField>

<ParamField path="NANOCLAW_SKIP" type="string">
  Comma-separated list of step names to skip during setup (debugging only). Equivalent flag: `--skip`.
</ParamField>

## Timezone configuration

The timezone is resolved through a priority chain:

1. `process.env.TZ`
2. `TZ` from `.env` file
3. `Intl.DateTimeFormat().resolvedOptions().timeZone` (system default)
4. `'UTC'` (fallback)

Each candidate is validated as a real IANA timezone identifier before being accepted. This affects cron expression evaluation for scheduled tasks.

## Directory paths

All paths are absolute and resolved from the project root:

<ResponseField name="DATA_DIR" type="string">
  `{PROJECT_ROOT}/data` — runtime data directory (central DB, sessions)
</ResponseField>

<ResponseField name="Central DB" type="string">
  `{PROJECT_ROOT}/data/v2.db` — central database with entity model
</ResponseField>

<ResponseField name="Sessions" type="string">
  `{PROJECT_ROOT}/data/v2-sessions/{agent_group_id}/{session_id}/` — per-session databases and files
</ResponseField>

<ResponseField name="GROUPS_DIR" type="string">
  `{PROJECT_ROOT}/groups` — agent group folders and memory files
</ResponseField>

<ResponseField name="MOUNT_ALLOWLIST_PATH" type="string">
  `~/.config/nanoclaw/mount-allowlist.json` — mount security allowlist (never mounted into containers)
</ResponseField>

## Trigger pattern

The default trigger pattern is generated from `ASSISTANT_NAME`:

```typescript
const DEFAULT_TRIGGER = `@${ASSISTANT_NAME}`;
const TRIGGER_PATTERN = new RegExp(`^${DEFAULT_TRIGGER}\\b`, 'i');
```

In v2, trigger behavior is controlled per-wiring via `engage_mode` and `engage_pattern` rather than a global trigger pattern.

## Example .env file

```bash
ASSISTANT_NAME=Andy
ASSISTANT_HAS_OWN_NUMBER=false
CONTAINER_TIMEOUT=1800000
MAX_CONCURRENT_CONTAINERS=5
TZ=America/Los_Angeles
ONECLI_URL=http://127.0.0.1:10254
```

<Note>
With the OneCLI Agent Vault, API keys and OAuth tokens are no longer stored in `.env`. Secrets are managed via `onecli secrets create` and injected by the vault at request time.
</Note>

## Credential management

Credentials are managed externally via OneCLI — no credential environment variables are needed in `.env`.

Register secrets with OneCLI:
```bash
onecli secrets create --name Anthropic --type anthropic --value YOUR_KEY --host-pattern api.anthropic.com
```

The `@onecli-sh/sdk` package's `applyContainerConfig()` configures each container's network to route through the vault, injecting credentials at request time.

## Security notes

- **Secrets** are never read by NanoClaw directly — OneCLI manages them externally
- The OneCLI Agent Vault injects credentials into container API traffic at request time
- Containers cannot extract real credentials from the vault
- Mount allowlist is stored outside the project root and never mounted into containers
- The `.env` file is read by the config module for NanoClaw settings only (not for API keys)
