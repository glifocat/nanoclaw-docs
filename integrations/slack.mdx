Add Slack as a messaging channel for NanoClaw. Perfect for team workspaces and professional communication.

## Overview

The Slack integration uses the Chat SDK bridge with HTTP webhooks. Slack delivers events to a public URL that NanoClaw exposes on port 3000 by default, so the host needs to be reachable from the internet (e.g. via ngrok, Cloudflare Tunnel, or a reverse proxy on a VPS).

### Features

- **Webhook delivery** - Events arrive via Slack's Event Subscriptions
- **Interactive components** - Approval cards and buttons via Interactivity & Shortcuts
- **Public and private channels** - Respond in any channel the bot is added to
- **Direct messages** - Users can DM the bot directly
- **Multi-channel support** - Monitor multiple channels simultaneously
- **Channel metadata sync** - Automatically syncs channel names
- **Thread flattening** - Reads threaded replies (responds in main channel)

## Adding Slack

Use the `/add-slack` skill to add Slack support. When run from inside `/setup`, the skill also wires the first agent to your Slack DMs and sends a welcome message.

<Steps>
  <Step title="Run the skill">
    ```bash
    claude
    /add-slack
    ```
  </Step>

  <Step title="Apply code changes">
    The skill copies the Slack adapter in from the `channels` branch:
    - `src/channels/slack.ts` (Chat SDK bridge registration)
    - Appends `import './slack.js';` to `src/channels/index.ts`
    - Installs `@chat-adapter/slack` (pinned version)
  </Step>

  <Step title="Create Slack app">
    Set up a Slack app with the required scopes, Event Subscriptions, and Interactivity & Shortcuts enabled
  </Step>

  <Step title="Configure environment">
    Add `SLACK_BOT_TOKEN` and `SLACK_SIGNING_SECRET` to `.env` and sync to the container
  </Step>

  <Step title="Wire your first agent">
    During `/setup`, the flow asks for your Slack member ID, opens a DM via `conversations.open`, and runs `init-first-agent` to wire the messaging agent. Outside `/setup`, run `/manage-channels` to wire an agent group.
  </Step>
</Steps>

## Creating a Slack app

The setup process requires a Bot Token (`xoxb-`) and a Signing Secret. The host must be publicly reachable so Slack can deliver events and interactive component payloads.

### Quick setup

<Steps>
  <Step title="Create the app">
    1. Go to [api.slack.com/apps](https://api.slack.com/apps)
    2. Click **Create New App** ŌåÆ **From scratch**
    3. Enter app name and select workspace
  </Step>

  <Step title="Add OAuth scopes">
    1. Go to **OAuth & Permissions**
    2. Add these **Bot Token Scopes**:
       - `chat:write` (send messages)
       - `im:write` (open DMs with users ŌĆö required for the welcome DM)
       - `channels:history` (read public channels)
       - `groups:history` (read private channels)
       - `im:history` (read DMs)
       - `channels:read` (list channels)
       - `groups:read` (list private channels)
       - `users:read` (look up display names)
       - `reactions:write` (send reactions)
  </Step>

  <Step title="Install to workspace">
    1. Click **Install to Workspace** and review permissions
    2. **Copy the Bot User OAuth Token** (starts with `xoxb-`)
    3. Go to **Basic Information** and copy the **Signing Secret**
  </Step>

  <Step title="Enable DMs">
    1. Go to **App Home** and enable the **Messages Tab**
    2. Check **"Allow users to send Slash commands and messages from the messages tab"**
  </Step>

  <Step title="Subscribe to events">
    1. Go to **Event Subscriptions** and toggle **Enable Events** on
    2. Set the **Request URL** to `https://your-domain/webhook/slack` ŌĆö Slack sends a verification challenge that must pass before you can save
    3. Under **Subscribe to bot events**, add:
       - `message.channels`, `message.groups`, `message.im`, `app_mention`
    4. Click **Save Changes**
  </Step>

  <Step title="Enable interactivity">
    1. Go to **Interactivity & Shortcuts** and toggle **Interactivity** on
    2. Set the **Request URL** to the same `https://your-domain/webhook/slack`
    3. Click **Save Changes**
    4. When Slack prompts you to **reinstall the app**, do it to apply the new settings
  </Step>
</Steps>

<Note>
Without **Interactivity & Shortcuts** enabled, button clicks on channel-approval cards silently fail ŌĆö Slack accepts them in the UI but never posts the payload to the webhook.
</Note>

  For detailed setup with screenshots, see the [SLACK_SETUP.md guide](https://github.com/qwibitai/NanoClaw/blob/main/.claude/skills/add-slack/SLACK_SETUP.md).

## Configuration

### Environment variables

Add to `.env`:

```bash
SLACK_BOT_TOKEN=xoxb-your-bot-token-here
SLACK_SIGNING_SECRET=your-signing-secret
```

The Chat SDK bridge starts a shared webhook server on port 3000 (override with `WEBHOOK_PORT`) and serves `/webhook/slack`. That port must be publicly reachable for Slack to deliver events.

### Credential reference

| Credential | Prefix | Where to find it |
|-------|--------|------------------|
| Bot User OAuth Token | `xoxb-` | **OAuth & Permissions** ŌåÆ **Bot User OAuth Token** |
| Signing Secret | _none_ | **Basic Information** ŌåÆ **App Credentials** ŌåÆ **Signing Secret** |

## Adding bot to channels

The bot only receives messages from channels it has been explicitly added to.

<Steps>
  <Step title="Open channel details">
    Click the channel name at the top of Slack
  </Step>
  
  <Step title="Add integration">
    Go to **Integrations** ŌåÆ **Add apps**
  </Step>
  
  <Step title="Search and add">
    Search for your bot name and add it to the channel
  </Step>
</Steps>

Repeat for each channel you want the bot to monitor.

## Registering channels

### Get channel ID

**Option A - From URL:**

Open the channel in Slack on the web. The URL looks like:

```
https://app.slack.com/client/TXXXXXXX/C0123456789
```

The `C0123456789` part is the channel ID.

**Option B - Right-click:**

Right-click the channel name ŌåÆ **Copy link** ŌåÆ extract the `C...` ID from the URL

**Option C - Via API:**

```bash
curl -s -H "Authorization: Bearer $SLACK_BOT_TOKEN" \
  "https://slack.com/api/conversations.list" | jq '.channels[] | {id, name}'
```

### Register the channel

The JID format is `slack:` followed by the channel ID.

For your main channel (responds to all messages):

```typescript
registerGroup("slack:C0123456789", {
  name: "#general",
  folder: "main",
  trigger: `@${ASSISTANT_NAME}`,
  added_at: new Date().toISOString(),
  requiresTrigger: false,
});
```

For additional channels (trigger-only):

```typescript
registerGroup("slack:C9876543210", {
  name: "#team-chat",
  folder: "team-chat",
  trigger: `@${ASSISTANT_NAME}`,
  added_at: new Date().toISOString(),
  requiresTrigger: true,
});
```

## Testing the connection

Send a message in your registered Slack channel:

- **For main channel:** Any message works
- **For non-main channels:** Include your trigger pattern (e.g., `@Andy hello`)

The bot should respond within a few seconds.

### Check logs

```bash
tail -f logs/nanoclaw.log
```

Look for:

```
Connected to Slack
Received message from slack:C0123456789
Agent response queued
```

## JID format

Slack channels use the `slack:` prefix:

- **Public channel:** `slack:C0123456789`
- **Private channel:** `slack:G0123456789`
- **Direct message:** `slack:D0123456789`

## Channel types

The Slack integration supports:

- **Public channels** - Bot must be added to the channel
- **Private channels** - Bot must be invited to the channel
- **Direct messages** - Users can DM the bot directly
- **Multi-channel** - Can monitor multiple channels simultaneously

## Implementation details

The Slack adapter is provided by `@chat-adapter/slack` via the Chat SDK bridge. Inbound events arrive on the shared webhook server at `/webhook/slack`, are verified using `SLACK_SIGNING_SECRET`, and are translated into the host's standard message envelope. Outbound messages ŌĆö including welcome DMs sent during setup ŌĆö go out through `chat.postMessage`.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Bot not responding">
    Check these items:
    
    1. Both tokens are set in `.env` AND synced to `data/env/env`
    2. Channel is registered:
       ```bash
       sqlite3 store/messages.db "SELECT * FROM registered_groups WHERE jid LIKE 'slack:%'"
       ```
    3. For non-main channels: message must include trigger pattern
    4. Service is running:
       ```bash
       launchctl list | grep nanoclaw
       ```
  </Accordion>
  
  <Accordion title="Bot not receiving messages">
    Verify these settings:

    1. Event Subscriptions is enabled and the Request URL passed Slack's verification challenge
    2. The webhook server (port 3000 by default) is reachable from the public internet
    3. Bot is subscribed to the correct events:
       - `message.channels`, `message.groups`, `message.im`, `app_mention`
    4. Bot has been added to the channel
    5. Bot has all the required OAuth scopes
  </Accordion>

  <Accordion title="Approval buttons do nothing">
    If channel-approval cards render but clicks have no effect, Interactivity & Shortcuts is not enabled. Toggle **Interactivity** on, set the Request URL to `https://your-domain/webhook/slack`, save, and reinstall the app when Slack prompts.
  </Accordion>
  
  <Accordion title="Missing scope errors">
    If logs show `missing_scope` errors:
    
    1. Go to **OAuth & Permissions** in Slack app settings
    2. Add the missing scope listed in the error
    3. **Reinstall the app** to workspace (scope changes require reinstall)
    4. Copy the new Bot Token (it changes on reinstall)
    5. Update `.env` and sync: `cp .env data/env/env`
    6. Restart: `launchctl kickstart -k gui/$(id -u)/com.nanoclaw`
  </Accordion>
  
  <Accordion title="Wrong token used">
    Verify the credential format:

    - Bot User OAuth Token starts with `xoxb-` (from **OAuth & Permissions**)
    - Signing Secret is a hex string with no prefix (from **Basic Information** ŌåÆ **App Credentials**)

    If `SLACK_BOT_TOKEN` doesn't start with `xoxb-`, you likely copied the App-Level Token (`xapp-`) by mistake.
  </Accordion>
</AccordionGroup>

## Message formatting

Slack uses [mrkdwn](https://api.slack.com/reference/surfaces/formatting) syntax, which differs from standard Markdown. NanoClaw includes a `/slack-formatting` container skill that provides the agent with a complete mrkdwn reference.

Key differences from standard Markdown:

- `*bold*` instead of `**bold**`
- `<url|text>` instead of `[text](url)`
- `ŌĆó` bullets instead of numbered lists
- No `##` headings ŌĆö use `*Bold text*` instead
- `:emoji:` shortcodes for emoji

The formatting rules are also embedded in the group `CLAUDE.md` files so agents always have access to them, even without invoking the skill directly.

<Tip>
If you apply the `/channel-formatting` skill, Markdown-to-mrkdwn conversion happens automatically in the outbound pipeline ŌĆö the agent doesn't need to know Slack's syntax at all. Links are converted to `<url|text>` format and bold markers are adjusted. See [channel-aware formatting](/features/messaging#channel-aware-formatting) for details.
</Tip>

## Known limitations

- **Threads are flattened** - Threaded replies appear as regular channel messages. Responses go to main channel, not back into threads.
- **No typing indicator** - Slack's Bot API doesn't expose a typing indicator endpoint.
- **Naive message splitting** - Long messages split at 4000 chars, may break mid-sentence.
- **No file handling** - Only text content is processed. File uploads and rich blocks are ignored.
- **Unbounded metadata sync** - Channel list pagination has no upper bound, may be slow in large workspaces.

## Removing Slack

Since Slack was added via a git merge, you remove it by reverting the merge commit:

<Steps>
  <Step title="Find the merge commit">
    ```bash
    git log --merges --oneline | grep slack
    ```

    Look for a commit like `abc1234 Merge remote-tracking branch 'upstream/skill/slack'`.
  </Step>

  <Step title="Revert the merge">
    ```bash
    git revert -m 1 <merge-commit>
    ```

    This creates a new commit that undoes all of Slack's code changes.
  </Step>

  <Step title="Remove registrations">
    ```bash
    sqlite3 store/messages.db "DELETE FROM registered_groups WHERE jid LIKE 'slack:%'"
    ```
  </Step>

  <Step title="Rebuild and restart">
    ```bash
    npm install
    npm run build
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw  # macOS
    # or: systemctl --user restart nanoclaw (Linux)
    ```
  </Step>
</Steps>

<Warning>
  If you later want to re-apply Slack after reverting, you must revert the revert first ŌĆö git treats reverted changes as "already applied and undone". Claude handles this automatically when you run `/add-slack` again.
</Warning>
