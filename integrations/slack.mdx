Add Slack as a messaging channel for NanoClaw. Perfect for team workspaces and professional communication.

<Note>
Slack is marked **experimental** in the setup picker. The adapter runs through the Chat SDK bridge and needs a public webhook URL before inbound events work — there is no welcome DM on install.
</Note>

## Overview

The Slack integration uses the Chat SDK bridge with the `@chat-adapter/slack` adapter and Slack's **Event Subscriptions** (webhooks). The bot receives events at `https://your-domain/webhook/slack`, which means you need a public URL — either a server, or a tunnel like ngrok or Cloudflare Tunnel.

### Features

- **Event Subscriptions** — Slack posts events to your webhook server
- **Public and private channels** — respond in any channel the bot is added to
- **Direct messages** — users can DM the bot directly
- **Multi-channel support** — monitor multiple channels simultaneously
- **Threads** — threaded replies are supported
- **Reactions** — bot can add reactions for typing / received / done signals

### Requirements

- A publicly reachable URL for `/webhook/slack` (a server, reverse proxy, ngrok, or Cloudflare Tunnel)
- A Slack workspace you can install apps into

## Adding Slack

Run `/setup` and pick **Yes, connect Slack (experimental)** at the channel prompt, or run the skill directly from Claude Code:

<Steps>
  <Step title="Run the skill">
    ```bash
    claude
    /add-slack
    ```
  </Step>

  <Step title="Create the Slack app">
    The skill walks you through creating an app at [api.slack.com/apps](https://api.slack.com/apps), adding OAuth scopes, enabling the Messages tab, and copying the Bot User OAuth Token + Signing Secret.
  </Step>

  <Step title="Paste tokens">
    Paste the `xoxb-` bot token and the signing secret. The skill validates them against `auth.test` and resolves your workspace + bot identity.
  </Step>

  <Step title="Install the adapter">
    The skill fetches `src/channels/slack.ts` from the `channels` branch, appends the self-registration import, installs `@chat-adapter/slack` (pinned), and rebuilds.
  </Step>

  <Step title="Configure Event Subscriptions">
    Set the **Request URL** in Slack to `https://your-domain/webhook/slack`, subscribe to bot events, and reinstall the app. The skill prints a post-install checklist with this step.
  </Step>

  <Step title="Wire an agent">
    Run `/manage-channels` to wire the Slack channel to an agent group. Slack does not auto-send a welcome DM — DM the bot yourself once the webhook is live to bootstrap the channel.
  </Step>
</Steps>

## Creating a Slack app

The setup process needs two values: a **Bot User OAuth Token** (`xoxb-`) and a **Signing Secret**.

<Steps>
  <Step title="Create the app">
    1. Go to [api.slack.com/apps](https://api.slack.com/apps)
    2. Click **Create New App** → **From scratch**
    3. Enter an app name and select a workspace
  </Step>

  <Step title="Add OAuth scopes">
    Go to **OAuth & Permissions** and add these **Bot Token Scopes**:

    - `chat:write` (send messages)
    - `channels:history` (read public channels)
    - `groups:history` (read private channels)
    - `im:history` (read DMs)
    - `channels:read` (list channels)
    - `groups:read` (list private channels)
    - `users:read` (look up display names)
    - `reactions:write` (typing/received/done reactions)
  </Step>

  <Step title="Enable the Messages tab">
    Go to **App Home** and:

    1. Enable the **Messages Tab**
    2. Check **Allow users to send Slash commands and messages from the messages tab**
  </Step>

  <Step title="Copy the signing secret">
    Go to **Basic Information** → **App Credentials** and copy the **Signing Secret**.
  </Step>

  <Step title="Install to workspace">
    1. Go to **Install App**
    2. Click **Install to Workspace** and review permissions
    3. Copy the **Bot User OAuth Token** (starts with `xoxb-`)
  </Step>

  <Step title="Enable Event Subscriptions">
    1. Go to **Event Subscriptions** and toggle **Enable Events**
    2. Set the **Request URL** to `https://your-domain/webhook/slack` — Slack sends a verification challenge that must pass before you can save
    3. Under **Subscribe to bot events**, add:
       - `message.channels`
       - `message.groups`
       - `message.im`
       - `app_mention`
    4. Click **Save Changes**
    5. Slack shows a banner asking you to **reinstall the app** — click it to apply the new event subscriptions
  </Step>
</Steps>

## Configuration

### Environment variables

Add to `.env`:

```bash
SLACK_BOT_TOKEN=xoxb-your-bot-token-here
SLACK_SIGNING_SECRET=your-signing-secret-here
```

### Token reference

| Value | Prefix | Where to find it |
|-------|--------|------------------|
| Bot User OAuth Token | `xoxb-` | **OAuth & Permissions** → **Bot User OAuth Token** |
| Signing Secret | — | **Basic Information** → **App Credentials** |

## Webhook server

The Chat SDK bridge starts a shared webhook server on port `3000` (override with the `WEBHOOK_PORT` env var). Slack and other webhook-based adapters share this server — the Slack handler lives at `/webhook/slack`.

This port must be reachable from the public internet so Slack can deliver events.

### Exposing the server

<Tabs>
  <Tab title="ngrok">
    ```bash
    ngrok http 3000
    ```

    Use the generated `https://*.ngrok.app` URL as the base for `https://your-domain/webhook/slack`.
  </Tab>

  <Tab title="Cloudflare Tunnel">
    ```bash
    cloudflared tunnel --url http://localhost:3000
    ```

    Use the assigned `trycloudflare.com` URL (or a named tunnel bound to your domain).
  </Tab>

  <Tab title="Reverse proxy / VPS">
    Point a reverse proxy (nginx, Caddy, Traefik) at `localhost:3000` on a server that's already publicly reachable. Use its public hostname as the webhook base.
  </Tab>
</Tabs>

## Adding the bot to channels

The bot only receives messages from channels it has been explicitly added to.

<Steps>
  <Step title="Open channel details">
    Click the channel name at the top of Slack.
  </Step>

  <Step title="Add integration">
    Go to **Integrations** → **Add apps**.
  </Step>

  <Step title="Search and add">
    Search for your bot name and add it to the channel.
  </Step>
</Steps>

Repeat for each channel you want the bot to monitor.

## Platform ID format

Slack uses the `slack:` prefix, matching Slack's conversation ID convention:

| Type | Format | Example |
|------|--------|---------|
| Public channel | `slack:C{id}` | `slack:C0123ABC` |
| Private channel | `slack:G{id}` | `slack:G0456DEF` |
| Direct message | `slack:D{id}` | `slack:D0ARWEBLV63` |

**Finding a channel ID:** Right-click the channel name → **View channel details** — the Channel ID is at the bottom. Or open the channel in the web client; the ID is the last segment of the URL.

## Testing the connection

1. DM the bot in Slack or `@mention` it in a channel the bot is a member of.
2. Check logs:

   ```bash
   tail -f logs/nanoclaw.log
   ```

   Look for:

   ```
   Connected to Slack
   Received message from slack:C0123456789
   Agent response queued
   ```

3. For non-primary channels: include the agent's trigger pattern (e.g., `@Andy hello`).

## Removing Slack

<Steps>
  <Step title="Revert the skill commit">
    ```bash
    git log --oneline | grep slack
    git revert <skill-commit-sha>
    ```

    Or manually: comment out `import './slack.js'` in `src/channels/index.ts`, remove `SLACK_BOT_TOKEN` and `SLACK_SIGNING_SECRET` from `.env`, then `pnpm uninstall @chat-adapter/slack`.
  </Step>

  <Step title="Rebuild and restart">
    ```bash
    pnpm install
    pnpm run build
    launchctl kickstart -k gui/$(id -u)/com.nanoclaw  # macOS
    # or: systemctl --user restart nanoclaw (Linux)
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Slack won't save the Request URL">
    Slack sends a verification challenge to your webhook before saving. If it fails:

    - Make sure the webhook server is running (`tail -f logs/nanoclaw.log` should show it started on port 3000).
    - Make sure the public URL actually reaches your machine — `curl https://your-domain/webhook/slack` should not time out.
    - Make sure `SLACK_SIGNING_SECRET` matches the one on **Basic Information**. Slack uses the signing secret to sign the challenge.
  </Accordion>

  <Accordion title="Bot not receiving messages">
    1. Event Subscriptions are enabled AND the app was reinstalled after adding events
    2. Bot is subscribed to the right events: `message.channels`, `message.groups`, `message.im`, `app_mention`
    3. Bot has been added to the channel
    4. Bot has the required OAuth scopes
    5. Public URL is still active (ngrok URLs change on restart unless you use a reserved domain)
  </Accordion>

  <Accordion title="Missing scope errors">
    If logs show `missing_scope` errors:

    1. Go to **OAuth & Permissions** in the Slack app settings
    2. Add the missing scope listed in the error
    3. **Reinstall the app** to workspace (scope changes require reinstall)
    4. Copy the new Bot Token — it changes on reinstall
    5. Update `.env` and sync: `cp .env data/env/env`
    6. Restart the service
  </Accordion>

  <Accordion title="Wrong token used">
    Bot tokens start with `xoxb-`. If you pasted a different prefix (`xoxp-`, `xapp-`, `xoxa-`), you copied the wrong token — go back to **OAuth & Permissions** → **Bot User OAuth Token**.
  </Accordion>
</AccordionGroup>

## Message formatting

Slack uses [mrkdwn](https://api.slack.com/reference/surfaces/formatting) syntax, which differs from standard Markdown. NanoClaw includes a `/slack-formatting` container skill that provides the agent with a complete mrkdwn reference.

Key differences from standard Markdown:

- `*bold*` instead of `**bold**`
- `<url|text>` instead of `[text](url)`
- `•` bullets instead of numbered lists
- No `##` headings — use `*Bold text*` instead
- `:emoji:` shortcodes for emoji

<Tip>
If you apply the `/channel-formatting` skill, Markdown-to-mrkdwn conversion happens automatically in the outbound pipeline — the agent doesn't need to know Slack's syntax at all. See [channel-aware formatting](/features/messaging#channel-aware-formatting) for details.
</Tip>
